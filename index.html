Resumo Git

Teoria

Versionamento de Código: O versionamento de código é o processo de criar novas versões de um código toda vez que existir uma mudança significativa nele. De maneira geral, todo projeto de desenvolvimento é feito por etapas, sendo que as funcionalidades são incrementadas aos poucos. Por isso, é preciso criar versões que possam ser retomadas sempre que necessário. Em outras palavras, o versionamento de código é como ter vários arquivos, sendo que cada um conta com melhorias em comparação com o anterior. Porém, em vez de ter que salvar muitos arquivos (o que é inviável para a computação e para a organização), são feitas essas versões, direto no código. Assim, enquanto o código está sendo desenvolvido, o programador vai criando as versões conforme melhora, muda ou implementa algo. Depois, é possível voltar às versões anteriores caso seja necessário. Por conta disso, o versionamento de código faz parte das boas práticas de programação e se torna importante para a organização do projeto, consegue contribuir com o trabalho em equipe e também criar um “backup” de todas as mudanças.

Git: O Git é uma ferramenta que permite gerenciar o código-fonte de aplicativos, sistemas e produtos. Ele mantém o histórico das modificações feitas nos arquivos, permitindo que você volte atrás e recupere o estado do sistema em qualquer momento. Imagine poder comparar as mudanças feitas hoje com as de um ano atrás, encontrar bugs e estudar otimizações. O Git torna isso possível.

GitHub: GitHub é uma plataforma de hospedagem de código-fonte e colaboração de desenvolvimento de software.

Comandos

git config -- ver comandos git.

git init -- inicializar repositório local. Você pode criar um reporsitório na sua máquina (mkdir new-project) e depois entrar dentro dele (cd new-project) e iniciar o reporsitório git init.

git add README.md ou touch README.md -- para criar um arquivo README

git status -- mostrar o status da área de preparação e o estado dos arquivos.

https://readme.so/pt -- Link para editar um arquivo README. Depois que você editar você pode copiar o que está lá e cola em seu arquivo README.

Docs GitHub -- Na parte da documentação do GitHub, que é escrevendo com o GitHub, tem exemplos de como editar esse arquivo, tipo cmo criar um título, um link ...
